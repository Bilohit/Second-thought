00 — THE PHILOSOPHY

Stop organizing. Start capturing.

Most tools force you to categorize the moment inspiration strikes. Second Thought removes the cognitive load of filing by automating the decision, organizing, and linking processes via a local reasoning pipeline. Instead of breaking your flow with folder navigation, tags, or naming conventions, it lets you capture raw input — text, voice, or images — and intelligently files it into your knowledge base. Think of it as a personal librarian that never interrupts you.

Why "Second Thought"?

Your first thought is the idea itself — raw, unfiltered, fleeting. The second thought — where it belongs, how to tag it, what it connects to, how you'll find it again — is handled entirely by the LLM. You drop the capture and move on. The pipeline does the thinking for you, not with you.

The system runs entirely on your machine. The only network calls are to your local Ollama instance and optional outbound fetches for URLs you explicitly capture. There is no cloud, no account, no telemetry, and no lock-in. Your vault is a portable folder of plain .md files — fully compatible with Obsidian, Logseq, or any text editor.

Designed to be unobtrusive, it respects your focus and prevents cross‑monitor DPI jumps by using two separate windows — one for the pill, one for the menu overlay.

03 — THE PIPELINE

Transparent execution, streamed live.

Driven by a Python FastAPI server and streamed via Server‑Sent Events (SSE). The modular design lets you add custom handlers for any data type. Every step logs its progress, giving you full visibility into what the system is doing — no opaque spinners, no black boxes.

1
Intercept
Normalizes clipboard / CLI input via interceptor.py. Handles text, URLs, images, and file paths uniformly into a single InputPayload.
↓
2
Pre‑resolve
Heuristically detects content shape (plain text / URL / GitHub / YouTube / image / audio) and suggests a category hint before the LLM is invoked.
↓
3
Enrich
Dispatches to specific handlers: web scraper + readability extraction for URLs, YouTube transcript fetch, local Whisper for audio, vision model for images. Fail‑soft for most types; fail‑fast for vision when required.
↓
4
Structure + Store
LLM generates a title, tags, backlinks, and picks the best folder based on your vault's existing organization. Two‑pass retry on parse failure. Dedup check against existing notes; merges or writes, or routes to scratchpad.
Pro tip

Customize the pipeline by editing pipeline.yml. Add your own handlers for PDFs, emails, or any domain‑specific data. The enrichment router is designed for extensibility — each handler is a drop‑in module.

Categories are never hardcoded. The category enum is rebuilt from your vault's current folder names on every single capture. Rename a folder, add a new one, or reorganize your vault — categorization adapts instantly, with zero configuration.

04 — FAILURE PHILOSOPHY

Fail gracefully. Never lose data.

Second Thought follows a clear failure philosophy: vision is fail‑fast; everything else is fail‑soft. The system is designed so that no single point of failure ever results in lost content. Here's what happens in every edge case:

SituationBehavior
Web fetch failsFail‑soft: a placeholder note is filed; you keep the capture.
YouTube transcript missingFail‑soft: placeholder; capture isn't lost.
YouTube summarization failsRaw transcript is already saved (write‑before‑summarize).
Audio transcription failsFail‑soft: placeholder with original metadata preserved.
Vision fails + image_required=trueFail‑fast: routed to scratchpad; never handed to the LLM with false confidence.
AI structured output won't parseTwo‑pass retry with stricter prompting; if it still fails → scratchpad.
AI low confidence on categoryRouted to the scratchpad — nothing is confidently mis‑filed.
Duplicate content detectedMerged into the existing note rather than creating a duplicate.
Pill dragged off‑screen / monitor unpluggedClamped back into visible bounds on next move.
You delete a derived databaseRebuilt from the vault — zero data loss. .md files are always authoritative.
Design principle

Write before summarize. For YouTube captures, the raw transcript is written to the vault immediately upon fetch. Summarization runs in the background afterward. Even if the summarizer crashes, your transcript is safe. The pill is free to use again right away — no waiting for long‑form processing.

05 — PRIVACY & DATA OWNERSHIP

Your data never leaves.

No Cloud, No Telemetry

The only network traffic is to your local Ollama instance. External fetches occur only when you capture a URL. No usage statistics, no phoning home. Air‑gap friendly — the system works fully offline for text, image, and audio captures.

Portable Markdown

Your vault is a plain folder of .md files with frontmatter. You own the data. Fully compatible with Obsidian, Logseq, and any text editor. No lock‑in, ever. Moving your vault is as simple as copying a folder and updating one config line.

Derived Indexes, Not Authorities

captures.db, vectors.db, and dedup_index.json are rebuildable caches over the vault. They never gate a merge, dedup, or link decision ahead of reading the actual .md files. Delete them anytime — they regenerate from your vault with zero data loss.

Local Whisper Transcription

Audio captured via --audio is transcribed entirely offline using openai-whisper + torch. Your voice memos never touch an external API. Multilingual, accurate, and fully private.

06 — INFRASTRUCTURE

Under the hood.

A lean three-tier architecture where each layer does one thing well. The Tauri shell handles the desktop experience, a FastAPI server drives the intelligence pipeline, and your Markdown vault remains the timeless source of truth — no proprietary databases, no lock-in.

Ollama

Powers all AI capabilities — embeddings, vision analysis, tagging, and structured output generation. Swap between any GGUF model in Settings. Runs entirely on your machine with zero external API calls.

Whisper

Transcribes voice memos offline using openai-whisper. Multilingual, highly accurate, and completely private. Optional OCR via rapidocr-onnxruntime extracts text from images before AI processing.

Tauri Shell

A lightweight Rust desktop shell that spawns the Python backend as a child process. Manages global hotkeys, the system tray, and the always-on-top pill window. Cross-platform with minimal resource footprint.

FastAPI + SSE

Every capture streams its progress live to the GUI via Server‑Sent Events. Watch each pipeline stage complete in real time — interception, enrichment, structuring, and storage — with full transparency into what the AI is doing.

Vector Store + FTS5

Dual search engines over your vault. SQLite-backed embeddings with cosine similarity for semantic search, plus FTS5 for precise full‑text queries. Both indexes are fully rebuildable from your Markdown files at any time.

Wikilink Resolution

Automatically discovers connections between notes by indexing frontmatter aliases across your entire vault. New captures are linked to existing related notes without any manual effort — your knowledge graph grows organically.

Map‑Reduce Summarizer

Handles articles and transcripts of any length by chunking content and applying token‑budget math. Long‑form summarization runs in the background so the pill stays responsive. You capture and move on; the summary appears when it's ready.

09 — GET STARTED

Begin in under two minutes.

Follow these steps to transform your fleeting thoughts into an organized knowledge base.

1

Install Ollama

Download from ollama.ai and pull a model, e.g. ollama pull mistral. For image captures, pull a vision-capable model. Make sure the Ollama service is running.

2

Install Python dependencies

Run pip install -r omni_capture/requirements.txt. This pulls in FastAPI, uvicorn, instructor, openai-whisper, torch, and other core dependencies.

3

Launch Second Thought

Windows: run .\launch.ps1. Development: npm run dev from gui/. On first launch, point it at a new or existing vault folder — this becomes your source of truth.

4

Set your hotkey & capture

Default is Ctrl+Shift+Space. Customize it in Settings. Copy text, drag a URL, or speak a note — then trigger a capture. The pipeline files it instantly. Check your Inbox to review.

Quick config check (terminal) 
# Ensure Ollama is up and has models available
                        curl http://localhost:11434/api/tags

                        # Run the self-check to verify everything
                        python main.py --self-check

                        # Start the full app (Windows)
                        .\launch.ps1

                        # Or run the dev version
                cargo tauri dev
Heads up

The Ollama base URL in config.toml must stay bare — just the host and port, e.g. http://localhost:11434 — never append /v1. The app adds that suffix internally only where needed. A /v1 in config breaks image captures with 404s.
