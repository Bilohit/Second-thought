01 · Color — 9 Themes via CSS Custom Properties

Every component reads var(--token) — never a raw hex. Swapping data-theme on <html> is the only thing that changes. Two grayscale themes lock the accent to neutral gray LOCKED; seven pastel themes get their own accent hue.

Radius scale — all 0px LOCKED

--radius-sm, --radius, --radius-lg, --radius-xl all resolve to 0px. One scale, defined once at :root, never redefined per theme. Components reference the named tokens (not a literal 0) so a future radius change only touches index.css.

Deliberate exception: PillOverlay.tsx's minimal-mode pill and its menu chips support a "rounded" corner style as a narrow, contained escape hatch — it touches only that one component tree, never panels or controls elsewhere. EXCEPTION

04 · Motion — Easing Curves & Durations

Two curve families, never mixed: a plain settle (--menu-travel-ease) for open/close travel, and an overshoot (--menu-overshoot-ease / --hover-ease) reserved for hover-only feedback. Hover splits enter/leave easing asymmetrically — see below.

Row language splits by intent: .row-hover-lift (lift + bg, no scale) for actionable cards like Inbox items; .row-hover-flat (bg only, no lift) for read-only rows inside scroll areas, so lists don't jitter while scrolling; .icon-close-btn is color-only on hover — a corner close button that "pops" reads as an accidental target.

06 · Legacy Config — Not Load-Bearing

gui/tailwind.config.js defines a violet accent (#7C6CF6) and a translucent glass palette (rgba(12,12,14,0.82)) that no component references — every surface in this codebase resolves color through the CSS custom properties in index.css instead. Tailwind's utility classes (@tailwind base/components/utilities) are still wired into index.css, but the custom theme extension is dead weight from an earlier design pass. Treat index.css as the single source of truth for color/radius/motion tokens.
